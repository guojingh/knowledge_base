# Vs code debug

**本文主要介绍在vs code中如何 debug golang代码**

## 1.准备环境

1. 安装 Golang，其中 Golang版本大于等于1.8

2. 安装 Golang 扩展

   1. 命令面板输入 Go:install/update tools

   2. 点击确定

      ![image-20250402111507358](https://picpoahu.oss-cn-chengdu.aliyuncs.com/images/image-20250402111507358.png)

   3. 这些工具虽然不是Go语言自带的，但它们是Go语言开发中广泛使用的工具，由Go社区和官方团队维护和开发。Vs Code的Go扩展通过管理这些工具来增强编辑器的功能，使开发者能够高效地进行开发工作。

      简单说明一下这些拓展文件的作用：

      1. gopls: Go语言服务器，用于提供智能代码补全，语法检测，重构等功能。
      2. gotests: 自动生成Go语言的测试代码。
      3. gomodifytags: 修改结构体字段中的标签，允许快速添加，删除或修改Go结构体字段的标签(如：json,xml标签等)
      4. impl: 生成接口等实现
      5. goplay: 在 Go Playground 上玉宁Go代码
      6. dlv: Delve是Go语言的调试器，支持代码断点，变量检查等调试功能。
      7. Staticcheck: 更高级的静态分析工具，提供代码优化建议和潜在的错误检查。

## 2.调试主程序

### 1. Run and Debug

默认情况下，最简单的方式开启对主程序的调试是，打开 main.go，启动 Run and Debug，此时vscode就能够自动识别对主程序的调试。

**注意：1.窗口打开main.go文件 2.打断点 3.点击Run and Debug**

![image-20250402114542936](https://picpoahu.oss-cn-chengdu.aliyuncs.com/images/image-20250402114542936.png)

### 2.借助 Launch 文件

说明：launch.json 文件的主要作用是提供一个结构化的配置方式，以便 vs code 中启动和管理不同类型的调试会话。它定义了许多调试相关的参数，如调试器的类型，运行参数，环境变量等，从而在调试过程中能够提供更多的控制和定制。

   下面是 launch.json文件的一些常用配置参数：

   1. **`"type"`**：指定调试器的类型，例如 "node" 表示 Node.js 调试器，"python" 表示 Python 调试器，"java" 表示 Java 调试器等。
   2. **`"request"`**：指定调试的请求类型，可以是 "launch"（启动一个新的进程）或 "attach"（附加到已有的进程）。
   3. **`"name"`**：为配置提供一个友好的名称，方便识别不同的调试配置。
   4. **`"program"`**：用于指定程序的入口文件路径，可以是绝对路径或相对于工作目录的路径。
   5. **`"args"`**：传递给程序的命令行参数，以数组形式提供。
   6. **`"cwd"`**：指定程序的工作目录，可以是绝对路径或相对于工作目录的路径。
   7. **`"env"`**：设置程序运行时的环境变量，以对象形式提供。
   8. **`"stopOnEntry"`**：设置为 **`true`** 时，在启动后会在入口处停止，等待调试器连接。
   9. **`"preLaunchTask"`**：指定在启动调试前运行的任务，通常是一个编译任务。
   10. **`"postDebugTask"`**：指定在调试结束后运行的任务，比如清理任务。
   11. **`"outFiles"`**：设置输出文件的路径，用于映射源代码和编译后的文件。
   12. **`"sourceMaps"`**：控制是否启用源代码映射，可以是 "inline"、"both" 或 "false"。
   13. **`"sourceMapPathOverrides"`**：用于根据源代码映射调整文件路径。
   14. **`"externalConsole"`**：设置为 **`true`** 时，将在外部控制台中运行程序。
   15. **`"internalConsoleOptions"`**：控制内部控制台的显示方式，可以是 "neverOpen"、"openOnSessionStart" 或 "openOnFirstSessionStart"。
   16. **`"showAsyncStacks"`**：设置为 **`true`** 时，在堆栈跟踪中显示异步调用的信息。
   17. **`"stopOnError"`**：设置为 **`true`** 时，当发生错误时暂停调试。
   18. **`"smartStep"`**：设置为 **`true`** 时，跳过无需调试的代码。
   19. **`"skipFiles"`**：指定不需要调试的文件或文件夹。
   20. **`"justMyCode"`**：设置为 **`true`** 时，只调试自己的代码。

   其中刚开始着重关注 program

   下面是一些占位符：

   | Variable                       | Description                                                  |
